2단계: 프로젝트 실행 점검

Spring Boot 애플리케이션이 정상 기동되는지 확인한 내용을 정리한 노트입니다.

1. 환경 개요

  • 패키지 루트: com.practice.basic
  • 주요 의존성:
    • spring-boot-starter-web – REST/MVC
    • spring-boot-starter-data-jpa – JPA 연동
    • com.h2database:h2 – 인메모리 DB
    • org.projectlombok:lombok – 보일러플레이트 제거
  • 설정 파일: src/main/resources/application.yml

2. 실행 절차

순서 명령 기대 로그 / 설명
1 ./mvnw clean (선택) Maven 래퍼 준비, 필요 시 이전 빌드 정리.
2 ./mvnw spring-boot:run Tomcat started on port(s): 8080 로그가 보이면 서버 기동 성공.
3 curl http://localhost:8080 Whitelabel Error Page는 루트 매핑 미구현 의미이므로 실행 성공.

3. 확인 사항

  • 로그에서 “Tomcat started on port(s): 8080” 확인.
  • 브라우저/터미널에서 http://localhost:8080 접속 시 Whitelabel 페이지 출력.
  • 추후 컨트롤러 구현으로 Whitelabel을 정상 응답으로 대체 예정.

4. 트러블슈팅 힌트

  • 포트 충돌 시: application.yml의 server.port 값을 변경.
  • YAML 파싱 에러: 들여쓰기(스페이스 2칸) 재확인.
  • 의존성 오류: pom.xml의 필수 의존성 선언 여부 확인.

5. 다음 단계 준비

  • 3단계: GET /hello 엔드포인트 구현 예정.
  • 패키지 구조 예시: controller, service, dto 등 분리.
  • 컨트롤러 추가 후 동일한 실행 절차로 검증 예정.
